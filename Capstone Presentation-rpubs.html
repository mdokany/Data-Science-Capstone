Context

This is the R presentation for the Data Science Capstone project. The project concludes the Data Science Specialization from Johns Hopkins University, offered through the Coursera platform.

The goal of this project is create a predictive text model as SwiftKey, the corporate partner in this capstone, builds with its smart keyboards. Given a large Text Corpus of documents, called a corpora, Natural Language Processing tools and techniques are used to perform statistical analysis, occurences counts and associations on monolingual or multilingual texts.

The data sets made available for this project are extracted from the Internet, with 3 distinctive sources: 1) News, 2) Blogs and 3) Twitter feeds, provided in 4 different languages. In the data analysis, since the provided files are quite large, only random subsets of the English American data sets are retained.

The goal of the project is to build a text prediction algorithm. More specifically, the algorithm will be used in a Shiny App to predict the next word of a user-typed sentence.

Algorithm

- The files have been split into training a testing data sets. - Numbers, punctuation, white spaces and non-ASCII characters have been removed. - All texts have been converted to lower case and English stop words removed. - Term matrices are built for uni-grams, bi-grams, tri-grams and quadri-grams. - Sparse terms from the matrices are removed in order to limit the model size. - Term matrices are converted to 4 data frames and columns are created for individual words. w1for 1st word, w2 for second word… - A sum column is added in each data frame to sum up the times a n-gram appears. - Data frames are sorted from most to least frequent n-grams for faster access. - For the next word prediction of a user-typed sentence, identical processing is performed to prepare the text. - Depending the number of words in the processed sentence, the 3, 2 or 1 final words are used to search within term matrices for quadri-grams, tri-grams or bi-grams. - In case records are found in quadri-grams, the 4th word of the most frequent quadri-gram is selected. If not, tri-grams then bi-grams are searched. - If nothing is found, the most frequent uni-gram is selected.

Shiny App

The Shiny App may be accessed through the following link: Next Word Prediction.

- The side bar panel consists of a text input widget where the use can type or paste a text. - Below the typed text, a submit button widget is available to send the sentence to the server for processing. - The main panel of the application consists of 3 widgets, to display the results. - The first widget repeats the sentence as typed or pasted by the user and before processing by the server. - The second widget displays the guessed word by the application using the algorithm described previously. - The third widget displays in a table style, the top 5 results as found by the algorithm, whether in quadri-grams, tri-grams, bi-grams or uni-grams.
